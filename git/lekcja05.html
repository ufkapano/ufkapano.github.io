Korzystanie z programu Git (5) - gałęzie zdalne

WPROWADZENIE

Synchronizacja repozytoriów polega na uzgodnieniu rewizji zawartych w poszczególnych gałęziach. Odbywa się to identycznie jak przy łączeniu gałęzi lokalnych. 

# Listowanie gałęzi.

$ git branch                  # gałęzie lokalne
* master

$ git branch -r               # lub --remotes, gałęzie zdalne
  remotes/origin/master

$ git branch -a               # lub --all, wszystkie gałęzie
* master
  remotes/origin/master

# Pelne nazwy (referencje do rewizji) to:
# ref/heads/master
# ref/remotes/origin/master

POBIERANIE GAŁĘZI ZDALNEJ

# git fetch [remote_name] [branch_name]

$ git fetch origin master     # typowe

# Łączenie pobranej gałęzi zdalnej.
# Może wystąpić konflikt, jeżeli dodawaliśmy rewizje lokalnie.

$ git merge origin/master
